Dataset definitions

D1 — PPI (D-SCRIPT)

All species/splits from D-SCRIPT, protein IDs resolved to sequences. (seq_a, seq_b, label), symmetric generic proteins.

D2 — AVIDa, no COVID

hIL6 + hTNFa, VHH paired with antigen sequences. (seq_a=VHH, seq_b=antigen, label) — asymmetric roles.

D3 — training pool

D1 ∪ D2, common schema tagged by pair_type/source_dataset. This is what D4's held-out sets are evaluated against.

D4 — held-out (2 axes)

ACE2 axis: each RBD mutant paired with human ACE2, ace2_bind label.
Antibody axis: each RBD mutant paired with each of 8 named antibodies, label flipped to 1=binds.

Held-out cleanliness

The central question for this design: does anything in either D4 held-out axis already appear in the D3 training pool? Checked by exact sequence overlap, in both directions, for both axes.

ACE2 axis — verified clean

ACE2 sequence present in D1? False. Present in D2? False. RBD mutant sequences overlapping D1: 0/19,132. Overlapping D2: 0/19,132. Zero overlap in both directions.

Antibody axis — verified clean

Antibody sequences (8 unique) overlapping D1/D2 training pool: 0 / 0. RBD mutant sequences (paired with antibodies) overlapping D1/D2: 0 / 0. This second held-out axis is equally clean.

Length distribution shift: D1's training proteins are capped at 800aa max, but D4's held-out ACE2 sequence is 805aa — 5 residues longer than anything seen in D1 training. A minor but real out-of-distribution point: the held-out set isn't only a new domain, it also sits just past the edge of the training length distribution.

8-antibody escape panel

VH/VL for all 8 named antibody clones (COV2-2050/2096/2094/2677/2479/2165/2499/2832) were unavailable from the MLAEP data itself, or from direct patent/PDB/GenBank search, but were found in CoV-AbDab (Oxford OPIG's curated coronavirus antibody database), correctly cited there to the original source (Zost et al. 2020, Nature Medicine). seq_b = VH + / + VL (explicit separator — VH and VL are two distinct polypeptide chains, not a fused construct). MLAEP's native COV2-*_400 columns are escape indicators (1 = antibody fails to bind); flipped here to 1 = binds for consistency with D1-D3.

Other cross-dataset facts

  • D1↔D2 overlap: exactly 1 sequence (human TNF-alpha, 233aa) appears in both D1 (as a generic PPI protein) and D2 (as the AVIDa-hTNFa antigen). Not a leakage concern since both are part of the same D3 training pool — just a notable cross-dataset subject-matter overlap.
  • D3 duplicate rows: 3,886 exact duplicate rows; 3,950 duplicate (seq_a, seq_b) pairs ignoring label — inherited mostly from D1's known ecoli_test.tsv duplicates, not a new issue introduced by combining.
  • Vocabulary (ACE2 axis): ACE2 non-standard residues: none. RBD mutants with non-standard residues: 0/19,132. Fully standard-alphabet.
  • Vocabulary (antibody axis): non-standard residues across all 8 VH/VL sequences (excluding the / separator): none. Fully standard-alphabet.

Notes

  • Column semantics differ by source within D3: in D1 (PPI), seq_a/seq_b are symmetric generic proteins; in D2 (AVIDa), seq_a is specifically the antibody (VHH) and seq_b is specifically the antigen — an asymmetric role. A model trained on D3 should be aware these columns don't mean the same thing across pair_type; consider an explicit role indicator if this matters for the chosen architecture.
  • D4's held-out sets have zero sequence-length variance (every RBD mutant is exactly 201aa; ACE2 is a constant 805aa; each antibody's VH/VL is likewise constant per clone) — the Phase-1 length-confound (positive fraction varying with pair length) mechanically cannot manifest in either held-out set, since there's no length variation to correlate with.
